信息网络安全第三章 Internet协议的安全性

一、网际层协议

1、IP协议

  • IP协议也称网际互连协议（Internet Protocol），它是TCP/IP协议族的核心，也是网际层最重要的协议。它主要负责数据包的传输和路由的选择。

  • IP协议的主要特点：

    • 无连接：各个数据报独立传输，可能沿着不同的路径到达目的地，也可能不会按序到达目的地。
    • 不可靠：不含错误检测或错误恢复的编码。
    • 点对点：进行数据传输时的对等实体一定是相邻设备或同一网络中的对等实体。
    • 效率高：采用了尽力传输的思想。
    • 无状态：通信双方不同步传输数据的状态信息，无法处理乱序和重复的IP数据包。

  • IP协议的主要功能：

    • 分配IP地址：IP协议为每个连接到互联网上的设备分配一个唯一的IP地址。
    • 路由选择：IP协议根据路由表选择最佳的路径将数据包传输到目的地。
    • 分组传输：IP协议将数据分成若干个数据报进行传输，每个数据包都有自己的目的地址和源地址。
    • 分片和重组：IP协议可以将大的数据包进行分片，传输到接收方后再进行重组。

2、ARP协议

  • 地址解析协议（Address Resolution Protocol）是根据IP地址获取物理地址的一个TCP/IP协议。
  • 安全问题：ARP欺骗攻击；
    防护措施：在交换机上配置IEEE 802.1x协议，攻击者连接交换机时需进行身份认证；或者建立静态ARP表。

3、ICMP协议

  • 控制报文协议（Internet Control Message Protocol）是一个重要的错误处理和信息处理协议。
    • 用于通知主机到达目的地的最佳路由，报告路由故障；
    • 是两个非常重要的监控工具——Ping和Tracert的重要组成部分；
    • 它是一种差错和控制报文协议，不仅用于传输差错报文，还传输控制报文。
  • 安全问题：ICMP重定向攻击
    • 攻击者可利用ICMP对消息重定向，使目标机遭受连接劫持和拒绝服务攻击。
    • 重定向消息应该仅由产生消息的主机或路由器执行，网管员不应使用ICMP创建通往目的地的新路由。
  • 安全问题：ICMP路由器发现攻击
    • 在进行路由发现时，ICMP并不对应答方进行认证，使它可能遭受中间人攻击。
  • 安全问题：防火墙穿越攻击
    • 通过防火墙穿越攻击技术，攻击者能够穿越某个防火墙的访问控制列表和规则集，进而确定该防火墙过滤的内容和具体的过滤方式。

4、IGMP协议

  • 组管理协议（Internet Group Management Protocol）是TCP/IP协议族中负责IP组播成员管理的协议。
    • 主机通过IGMP通知路由器希望接收或离开某个特定组播组的信息。
    • 路由器通过IGMP周期性地查询局域网内的组播组成员是否处于活动状态，实现所连网段组成员关系的收集与维护。

5、OSPF协议

  • 开放最短路径优先协议（Open Shortest Path First）是一个内部网关协议，用于在单一自治系统内实施路由决策。

  • OSPF路由协议是一种典型的链路状态的路由协议，一般用于同一个路由域内，属于分布式的链路状态路由协议。

  • 每个OSPF路由器都有一个保存全网链路信息的链路状态数据库。OSPF路由器正是通过这个数据库计算出其OSPF路由表。

  • OSPF协议中规定了认证域，但其作用非常有限。

6、BGP协议

  • 边界网关协议（Border Gateway Protocol）将单一管理的网络转化为由多个自治系统分散互连的网络。
  • 安全问题：BGP缺乏一个安全可信的路由认证机制，无法对所传播的路由信息的安全性进行验证。

二、传输层协议

1、TCP协议

  • 传输控制协议（Transmission Control Protocol），是一种面向连接的、可靠的传输层通信协议。
  • 第一类攻击：攻击前，针对TCP连接建立阶段的三次握手过程的SYN FLOOD攻击。攻击者不断向服务器的监听接口发送建立TCP连接的请求SYN数据包，使服务器一直处于半开放连接状态，无法接受其他正常的连接请求。属于拒绝服务攻击。
  • 第二类攻击：针对TCP协议不对数据包加密和认证的漏洞的TCP会话劫持攻击。
  • 第三类攻击：针对TCP协议的拥塞控制机制的特性，在连接建立后的数据传输阶段发起攻击。

2、UDP协议

  • 用户数据包协议（User Datagram Protocol）是一种无连接的传输层协议，提供面向事务的简单不可靠信息传送服务。
  • 安全问题：最常见的UDP攻击为Dos攻击，而UDP FLOOD攻击又是Dos攻击中最普遍的流量型攻击。攻击源发送大量的UDP小包到攻击目标，目标可以是服务器或网络设备，使其忙于处理和回应UDP报文，系统资源使用率高，最后导致设备不能提供正常服务或死机。

三、应用层协议

1、RIP协议（端口：UDP 520）

  • 动态路由选择协议（Routing Information Protocol）是一种动态内部路由/网关协议，用于自治系统内的路由信息的传递。
  • 安全问题：攻击者可以伪造RIP路由更新信息；利用网络嗅探工具来获得远程网络的RIP路由表，通过欺骗工具伪造RIP报文，再利用重定向工具截取、修改和重写向外发送的报文。

2、HTTP协议（端口：TCP 80）

  • 超文本传输协议（Hyper Text Transfer Protocol）是一种应用层协议，它规定了客户端和服务器端之间请求和应答的标准，也是互联网上应用最广泛的网络协议。
  • HTTP协议中数据是直接通过明文进行传输的，不提供数据加密，因此存在较大的安全缺陷。

3、TElNET协议（端口：TCP 23）

  • 远程登录协议（Teletype Network）是在Internet上实现远程登录服务的标准协议和主要方式。
  • TElNET提供了简单终端到某台主机的访问，攻击者可以通过Sniffer记录用户名和口令组合，或者记录整个会话。

4、SSH协议（端口TCP 22）

  • 安全壳协议（Secure Shell）是一种在不安全的网络上建立安全的远程登录网络服务的协议，支持身份认证和数据加密。
  • SSH协议服务器的主机密钥存储在一个主机密钥文件中。

5、DNS协议（端口UDP 53）

  • 域名系统（Domain Name System）是一个分布式数据库系统，用来实现域名与IP地址之间的映射。1个为主根服务器，放置在美国，其余12个均为辅根服务器，其中9个在美国，欧洲2个，位于英国和瑞典，亚洲1个，位于日本。
  • DNS欺骗攻击：攻击者假冒域名服务器的一种欺骗行为。
  • DNS缓存中毒攻击：攻击者给DNS服务器注入非法网络域名地址。
  • DNS重定向攻击：攻击者将DNS名称查询重定向到恶意DNS服务器上。

6、SMTP协议（端口：TCP 25）

  • 简单邮件传输协议（Simple Mail Transfer Protocol）是一组用于由源地址到目的地址传送邮件的规则，由它来控制信件的中转方式。
  • SMTP自身是完全无害的，但它可能成为拒绝服务攻击的发源地。

7、MIME协议

  • 多用途网间邮件扩充协议（Multipurpose Internet E-mail Extention）是一种广泛应用的电子邮件技术规范。
  • 对MIME存在一种分段攻击。
  • 邮递可执行程序或邮件含有危险的PostScript文件，是传播蠕虫和病毒的主要途径。

8、POP3协议（端口：TCP 110）

  • 邮局协议（Post Office Protocol）是一个邮件协议，它的第3个版本称为POP3。
  • POP3口令以明文传输，存储在服务器上。

9、IMAP4协议（端口：TCP 143,993）

  • 消息访问协议（Internet Message Access Protocol）是一个邮件接收协议。
  • IMAP4与POP3一样，是收邮件协议。

10、PGP协议

  • PGP（Pretty Good Privacy）协议是常用的安全电子邮件标准。PGP包括5种服务：认证、加密、压缩、电子邮件兼容和分段。

11、FTP协议（端口：TCP 20/21）

  • 文件传输协议（File Transfer Protocol）是Internet文件传送的基础，是TCP/IP协议族的重要协议之一。
    • 控制连接：在整个会话期间一直保持打开，客户发送的传送请求通过控制连接发送给服务器端的控制进程。
    • 数据连接：用于传输文件。

12、TFTP协议（端口：UDP 69）

  • 简单文件传输协议（Trivial File Transfer Protocol）是用来在客户机与服务器之间进行简单文件传输的协议。
  • TFTP用于提供简单、开销较小的文件传输服务，利用UDP协议，提供不可靠的数据流传输服务。

13、NFS协议（端口：UDP 2049）

  • 网络文件系统（Network File System）是一个基于TCP/IP网络的文件共享协议。
  • NFS允许一个系统在网络上与他人共享目录和文件。

14、SNMP协议（端口：UDP 161/162）

  • 简单网络管理协议（Simple Network Management Protocol）是用于支持网络管理系统的协议。
  • 广泛应用于多种类的网络设备、软件和系统中，被认为是首选管理协议。
  • 控制路由器、网桥及其其他网元，用来读/写各种设备信息，如操作系统版本、路由表、接口名称和ARP映射表等。

15、DHCP协议（端口：UDP 67/68）

  • 动态主机配置协议（Dynamic Host Configuration Protocol）用于分配IP地址、提供启动计算机其他信息的协议。
  • 出于安全性的考虑，此协议只能在本地网络上使用。

16、H.323协议

  • H.323是一种用于VoIP（Voice over IP）在分组网上提供实时音频、视频和数据通信的标准。

17、SIP协议

  • 会话启动协议（Session Initiation Protocol）是由IETF制定的基于文本编码的多媒体通信协议。

18、NTP协议（端口：UDP 123）

  • 网络时间协议（Network Time Protocol）可在分布式时间服务器和客户端之间进行时间同步。

19、FINGER协议（端口：UDP 79）

  • Finger协议可以帮助用户查询系统中某一个用户的细节，如其全名、住址、电话号码、登录细节等。

20、Whois协议（端口：TCP 43）

  • Whois协议是用来查询域名的IP以及所有者等信息的传输协议。

21、LDAP协议（端口：TCP 389）

  • 轻量级目录访问协议（Lightweight Directory Access Protocol）是基于X.500的目录访问协议，以目录的形式来管理资源。
  • 用于数字证书的存取。

22、NNTP协议（端口：TCP 119）

  • 网络新闻传输协议（Network News Transfer Protocol）主要用于阅读和张贴新闻文章到Usenet上的Internet应用协议。
