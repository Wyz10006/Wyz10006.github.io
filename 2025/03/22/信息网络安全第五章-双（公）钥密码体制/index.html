信息网络安全第五章 双（公）钥密码体制

一、双钥密码体制的基本概念

1、陷门单向函数

  • 单射：令函数 f 是集A到集B的映射，用 f : A→B表示。若对于任意x1≠x2，x1, x2∈A，有f (x1) ≠ f (x2)，则称 f 为单射，或1-1映射，或可逆的函数。
  • 单向函数：一个可逆函数 f : A→B，若它满足： (1) 对所有x ∈A，易于计算 f (x)； (2) 对“几乎所有 x ∈A”，由 f (x)求 x 极为困难，以至 于几乎是不可能的，则称 f 是一个单向函数。
  • 陷门单向函数：fz: Az→Bz，z∈Z，Z是陷门信息集合。
    • 对所有 z∈Z，在给定 z 下容易找到一对算法 Ez和 Dz，使对所有x∈A，易于计算 fz及其逆，即：fz(x)=Ez(x)f_z(x) = E_z(x)fz​(x)=Ez​(x)，Dz(fz(x))=xD_z(f_z(x)) = xDz​(fz​(x))=x
    • 对所有z∈Z，当只给定 Ez和 Dz 时，对所有 x∈A，很难从y=fz(x)计算出 x 。

2、公钥密码体制

  • 公钥密码体制，又称非对称密码体制或双钥密码体制 (Public-key/Two-key/Asymmetric Cryptosystem)，其加密和解密数据使用不同的密钥。

  • 每个用户都拥有两个密钥：公钥和私钥

    • 公钥(public-key)：可以被任何人知道，用于加密或验证签名；
    • 私钥(private-key)：只能由持有者知道，用于解密或签名。
    • 由私钥及其他密码信息容易计算出公钥；而由公钥及算法描述，计算私钥却非常困难。

  • 公钥密码体制解决了密钥的发布和管理问题

    • 通信双方可以公开其公钥，而保留私钥。
    • 发方可以用收方公钥对发送的信息进行加密。
    • 收方用自己的私钥对收到的密文进行解密

  • 公钥密码体制的用途

    • 密钥分发：用于交换对称加密密钥。
    • 消息加密：用公钥加密，用私钥解密；
    • 数字签名：发送方采用自己的私钥对消息进行签名；接收方采用发送方的公钥对签名进行验证。

  • 公钥密码体制的安全性

    • 安全性依赖于解数学上的困难问题。
      • 多项式求根、离散对数DL、大整数分解FAC、Diffie-Hellman问题(DHP)
    • 目前，通常要求足够大的密钥长度(>1024 bits) 。
    • 密钥太长会导致加密速度缓慢，因此公钥算法常用于密钥传递，而一般不用于实时数据加密。

二、RSA密码体制

1、RSA公钥算法的特点

  • RSA理论基础是一种特殊的可逆模指数运算，其安全性基于大整数分解的困难性。
  • 既可以用于消息加密，也可用于数字签名。
  • 硬件实现时比DES慢约1000倍。软件实现时比DES慢约100倍。

2、RSA公钥算法详解

  • 设 n 是两个不同奇素数之积，即 𝑛 = 𝑝 × 𝑞，计算其欧拉函数值 𝜑(𝑛) = (𝑝 − 1)(𝑞 − 1)。
  • 随机选一整数𝑒, 1 ≤ 𝑒 < 𝜑(𝑛) , gcd(𝜑(𝑛), 𝑒) = 1 ，因而在模 𝜑(𝑛)下, 𝑒有逆元：𝑑 = 𝑒−1 mod 𝜑(𝑛)（费马小定理）
  • 取公钥为𝑛, 𝑒, 私钥为𝑑 ( p, q不再需要，可销毁，但绝不可泄露)。
    • 加密变换为：c=memod n
    • 解密变换为：m=cdmod n

3、RSA算法的参数选择

  • 模数 𝑛 = 𝑝.𝑞 大于1024bit
  • p, q为大素数
  • p-1，q-1有大的素因子
  • p+1，q+1也要有大的素因子
  • e不能太小，例如 e 值为 (216+1) = 65537
  • e 必须要与 𝜑(𝑛)互素

4、RSA算法的局限性

  • 计算效率低，加解密速度慢，不适合加密大量数据
  • 密钥长度需求高，存储资源和传输开销大
  • 存在确定性加密的弱点
    • 为了增强安全性，RSA通常需要结合随机化技术（如OAEP填充）。

三、ElGamal密码体制

1、算法描述

  • 其安全性依赖于离散对数问题（discrete logarithms problem）
  • 选择素数 p，设GF(p)上的本原元为 g
  • 选择私钥： α in GF(p)*（except 0）
  • 计算公钥：β = gα mod p
  • 选择一个随机数：k，且(𝑘, 𝑝 − 1) = 1
  • 加密：m → (gᵏ , mβᵏ) mod p = (y₁, y₂) = c
  • 解密：m = y₂(y₁)⁻ᵃ mod p

2、安全性讨论

  • ElGamal的安全性基于有限域中离散对数问题的难解性。
  • 通常使用的素数 p 的长度为2048位或更长，以确保足够的安全性；安全性与RSA相当，但ElGamal的密文长度是明文的两倍，效率较低。
  • 在加密过程中引入了随机数，使相同的明文每次加密后生成的密文都不同；这一密文的随机化特性增强了其安全性，防止了确定性加密的弱点。

四、椭圆曲线密码体制

1、实数域上的椭圆曲线

  • 实数域上的椭圆曲线可以定义为满足方程：y² = x³ + ax + b 的所有点（x，y）的集合

    • 注意：椭圆曲线并不是椭圆，只因为该方程与计算椭圆周长的方程相似。

  • 可以证明：如果 x³ + ax + b 没有重复因子，或者满足 4a³ + 27b² ≠ 0，那么椭圆曲线上的点集 E(a, b) 可构成一个 Abel 群。

  • 椭圆曲线群包括所有曲线上点以及一个特殊的点，我们称其为无限远点 O。

  • 群定义：

    • 在集合上定义某种运算
    • 运算满足封闭性
    • 满足结合律
    • 有单位元和逆元。
      满足以上条件，我们就称这个集合为群。
    • 若还满足交换律，则构成 Abel 群。

  • 实数域椭圆曲线上加法：P+Q

  • 能找到𝑃的逆元−𝑃，使得 𝑃 + (−𝑃) = 𝒪（无穷远点）

  • 加法满足封闭性：𝟐𝑷 = 𝑷 + 𝑷 = R

2、有限域Ep(a, b)上的椭圆曲线

  • 有限域 GF(p) 上的椭圆曲线 y² = x³ + ax + b，其点集 (x, y) 构成有限域上的 Abel 群，记为 Ep(a, b)。条件为4a³ + 27b² ≠ 0 mod p

  • 设 P = (x₁, y₁)，Q = (x₂, y₂)，P + Q = (x₃, y₃)

    • 当 P ≠ Q 时：

      • λ = (y₂ - y₁) / (x₂ - x₁) mod p
      • x₃ = (λ² - x₁ - x₂) mod p
      • y₃ = (λ(x₁ - x₃) - y₁) mod p

    • 当 P = Q 时：

      • λ = (3x₁² + a) / (2y₁) mod p
      • x₃ = (λ² - x₁ - x₂) mod p
      • y₃ = (λ(x₁ - x₃) - y₁) mod p

3、椭圆曲线密码

  • 椭圆曲线上的离散对数问题——ECDLP：在椭圆曲线构成的 Abel 群 Ep(a, b) 上，考虑方程：Q = kP，其中 P, Q ∈ Ep(a, b)，k < p，由 k 和 P 计算 Q 非常容易，而由 P, Q 计算 k 则非常困难。

  • 椭圆曲线上的 Diffie-Hellman 密钥交换

    • A 选择一个小 n 的整数 nA 作为私钥，由 PA = nA G 产生 Ep(a, b) 上的一个点作为公钥。

    • B 选取自己的私钥 nB，并计算自己的公钥 PB = nB G。

    • A 可以获得 B 的公钥 PB。

    • B 可以获得 A 的公钥 PA。

    • A 计算：K = nA × PB = nA nB G

    • B 计算：K = nB × PA = nB nAG

    • 至此，A 和 B 共同拥有密钥 K = nA nB G。

  • 椭圆曲线公钥加密/解密算法

    • 准备阶段

      • A 选择一个小 n 的整数 nA 作为私钥，由 PA = nAG 产生 Ep(a, b) 上的一个点作为公钥。
      • B 选取自己的私钥 nB，并计算自己的公钥 PB = nBG。

    • 加密阶段

      • 若 A 要将消息 m 加密后发给 B，则 A 选择一个随机数 k，计算：C = {C₁, C₂} = {kG, m + kPB}

    • 解密阶段

      • B 收到密文 C 后，需用 C₂ 减去 C₁ 与 B 的私钥之乘积：m = (m + kPB) - nB(kG) = m + k(nBG) - nB(kG)

  • 相比于RSA密码体制，椭圆曲线密码体制最大的优势在于可以使用比RSA更短的密钥来获得相同水平的安全性，从而使计算量大大减少。
