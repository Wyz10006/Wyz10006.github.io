信息网络安全第四章 单（私）钥密码体制

一、密码体制的定义

  • 保密学(Cryptology)：包括密码编码学和密码分析学。
  • 密码编码学(Cryptography)：研究的是对数据进行变换的原理、手段和方法的技术和科学，旨在设计安全的密码体制。
  • 密码分析学(Cryptanalysis)：研究如何攻破一个密码系统，恢复被隐藏信息。它是在未知密钥的情况下从密文推演出明文或密钥的技术和科学。

1、密码学基本概念

  • 明文：需要秘密传送的可以读得懂的消息。
  • 密文：明文经过密码变换后的不可读消息。
  • 加密：由明文到密文的数学变换。
  • 解密：从密文恢复出明文的数学变换。
  • 加密算法：对明文进行加密时采用的一组规则。
  • 解密算法：对密文进行解密时采用的一组规则。
  • 密钥：加密和解密时使用的一组秘密信息。

2、密码体制的语法定义

  • 明文消息空间 M：可能的明文字母串集合

  • 密文消息空间 C：可能的密文字母串集合

  • 加密密钥空间 K：可能的加密密钥集合

  • 解密密钥空间 K’：可能的解密密钥集合

  • 有效的密钥生成算法 ζ：N → K × K’

  • 加密算法 E：M × K → C

  • 解密算法 D：C × K’ → M

  • 密钥生成算法：∀整数 1l，ζ(1l)输出长为 l 的密钥对：(ke, kd) ∈ K × K’

  • 加密/解密过程：

    • ∀m∈M, c∈C

    • c = Eke(m)

    • m = Dkd© = Dkd(Eke(m))

3、密码体制的分类

  • 若 ke = kd，则称为单钥加密体制（对称加密体制或私钥加密体制）
  • 若 ke ≠ kd，则称为双钥加密体制（非对称加密体制或公钥加密体制）

柯克霍夫准则：荷兰密码学家 Kerchoffs 于 1883 年提出，一个密码系统的安全性应完全依赖密钥的保密性，而不应该依赖于算法的保密。

4、单钥密码体制的分类

按照加解密运算的特点，分为：

  • 流密码(Stream Cipher)：数据逐比特加密，即数据流与密钥流逐比特进行异或运算。
  • 分组密码(Block Cipher)：对数据分组进行处理。

二、古典密码

古典密码学主要有两种基本方法：

  • 置换密码：明文字母保持不变，但顺序被打乱。
  • 代换密码：明文字母被替换，但顺序保持不变。
    • 单表代换：一个明文字母对应的密文字母是确定的。
    • 多表代换：一个明文字母可以表示为多个密文字母。
    • 弗纳姆密码：将每消息比特和相应的密钥比特进行比特异或运算。

1、代换密码

  • 单表代换：将26个字母用十进制数0~25表示，使用字母代换表将明文字母替换为密文字母。
    • 凯撒密码(Caesar Cipher)：将字母按顺序推后3位。
  • 多表代换：
    • 维吉尼亚密码：对英文字母表进行编号，把a ~ z依次编为0 ~ 25。计算明文字母编号与密钥字母编号的模26加法，得到密文字母。
  • 弗纳姆密码（Vernam Cipher）：将明文与密钥进行模2加法运算。如果M=C=K={0,1}*，则弗纳姆密码就是代换密码的特例；如果密钥串只使用一次，那么弗纳姆密码就是一次一密密码。也可以看成序列密码（流密码）的雏形。

2、置换密码

  • 置换密码（Permutation Cipher）又称换位密码，是通过重新排列明文中元素的位置而不改变元素本身来实现加密的体制，它广泛应用于现代分组密码的构造。
  • 置换就是简单的换位，每次置换都可以用一个置换矩阵来表示，且都有一个与之对应的逆置换矩阵。

3、古典密码的安全性

  • 穷举搜索攻击：通过尝试所有可能的密钥组合来破解密码。由于古典密码学中密钥空间较小，这种方法在计算资源有限的情况下是可行的。
  • 频率分析：通过分析密文中字母的频率来破解密码。例如，在英文文本中，字母 E 的出现频率最高，如果密文中某个字母的出现频率异常，可能暗示密钥的某些部分。
  • 差分攻击：通过分析密文中相邻字符的变化规律来破解密码。
  • 线性分析：通过分析密文中字符之间的线性关系来破解密码。

三、流密码

1、基本概念

  • 明文 m = m1, m2, …, ml
  • 伪随机序列 k = k1, k2, …, kl
  • 密文 ci = Eki(mi) = mi ⊕ ki
  • 解密过程与加密过程相同且互逆
  • 流密码的安全性完全依赖于伪随机数的强度

2、同步流密码

  • 若密钥流生成器内部状态与输入的明文消息无关，则密钥流将独立于明文，称此类流密码为同步流密码（Synchronous Stream Cipher）。

  • 对于明文而言，这类加密变换是无记忆的，但它是时变的。

  • 缺点：对失步敏感，只有保持两端精确同步才能够正常工作。

  • 优点：没有差错传播

3、自同步流密码

  • 若密钥流生成器内部状态与明文有关，则密钥流与明文 m = m1, m2, …, mi-1 有关，称这类流密码为自同步流密码（Self-Synchronous Stream Cipher）。

  • 缺点：传输过程中一位出错，将影响后 n 位密钥的正确性

  • 优点：收端只要连续正确地收到 n 位密文，则在同一个密钥 ki 作用下就会产生相同的密钥序列，因而它具有自同步能力。

四、分组密码概述

1、分组密码的概念

分组密码(Block Cipher)将明文消息编码表示后的明文数字序列，划分成长为m比特的分组 x = (x0, x1, …, xm-1) ，各分组分别在密钥 k = (k0, k1, …, kt-1) 的 控制下，变换成长度为n比特的密文输出序列 y = (y0, y1, …, yn-1) ，加密函数为单钥密码算法E: Vm × K→Vn , 其中Vm和Vn 是m、n 维矢量空间，K 为密钥空间。

2、分组密码的设计要求

  • 分组长度足够大（≥128~256比特）
  • 密钥量要足够大（≥128192256比特）
  • 算法足够复杂（包括子密钥产生算法）
  • 加密、解密算法简单，容易采用软、硬件实现
  • 数据无扩展（只在引入同态置换和随机化加密时有扩展）
  • 差错传播尽可能得小

3、Shannon的密码设计思想

  • 扩散(diffusion)：将每位明文及密钥数字的影响尽可能迅速地散布到较多个输出的密文数字中，以便隐蔽明文数字的统计特性。产生扩散的最简单方法是通过“置换(Permutation)”（比如：重新排列字符）。
  • 混淆(confusion)：目的在于使明文的密钥和密文之间的关系复杂化，使明文和密文之间、密文和密钥之间的统计相关特性极小化，从而使统计分析攻击不能奏效。通常的方法是“代换（Substitution)” 。

五、数据加密标准

1、DES介绍

  • DES: Data Encryption Standard是一种分组密码，其输入的明文长度为64bit，密钥长度为56bit，输出的密文长度为64bit。
  • 公布DES的意义：在密码史上首次践行Kerckhoffs准则，另外一个重要启示就是密码算法可以标准化，通信各方可采用相同的标准密码算法。
  • 扩散的实现，一般采用线性变换、置换、循环移位等手段。初始置换过程即为香农信息论中的“扩散”。
  • 混淆的实现，一般采用非线性的操作，例如乘法、S盒等。迭代运算过程包括香农信息论中的“混淆+扩散” 。
  • DES采用Feistel结构。Feistel 密码结构是一种对称结构，满足对合性。采用此结构的密码，其好处是加解密可以使用同一个芯片。
  • E盒的作用只是简单地将32bit数据分组扩展为48bit数据分组。
  • S盒的作用是混淆 ( Confusion )，是一种非线性变换，它将E表输出的48bit数据分组压缩成32bit数据分组，以增加明文和密文之间的复杂度（包括非线性度等）。DES的安全性直接取决于S盒的安全性。
  • P盒的作用是扩散 ( Diffusion )，这是一种线性变换，目的是让明文和密钥的影响迅速扩散到整个密文中。即一位的明文或密钥的改变会影响到密文的多个比特。

六、高级加密标准

1、Rijndael算法概述

  • Rijndael是分组长度和密钥长度均可变的分组密码，其密钥长度和分组长度可以独立指定为128/192/256bit。
  • AES的密钥长度为128/192/256bit，但分组长度固定为128bit。
  • 解密运算：Rijndael的4个内部函数都是可逆的，因此解密仅仅是在相反的方向反演加密。
  • Rijndael算法特点: (1)不是Feistel结构密码，属于SP网络 ；(2)加密和解密相似但不对称，必须分别使用不同的电路和代码；(3)结构简单，速度快。

2、AES密码算法

  • 成功取代了DES、3DES、IDEA等旧的分组密码算法。
  • 随着 AES 算法所采用的密钥长度不同 ，其加密轮数也发生变化 ，128/192/256b密钥对应的加密轮数为10/12/14。
  • 避免了多重加密（例如3DES）的使用，减少了实际应用中的密钥数量，可以简化安全协议和系统的设计。

七、分组密码的工作模式

1、电码本模式（ECB, Electronic Code Book）

  • 优点：
    • 无错误传播。若密文有1bit错误，仅会影响当前密文分组的正确解密，但错误不会传播给下一个分组，即不会影响其他分组的正确解密。
    • 可采用并行计算。ECB实现比较简单，因分组之间无关联，从性能的角度看有些优势。
  • 缺点：
    • 是确定性加密模式，若在相同的密钥作用下，将P1,P2,…,Pm加密2次，则输出的密文分组也是相同的。如果明文消息是可猜测的，那么攻击者就会用试凑法猜测出明文。
    • 缺乏随机性。每个分组独立加密，不能有效抵抗统计攻击。因为攻击者可以通过分析密文分组之间的统计特性来推断出一些明文信息。

2、密码分组链接模式（CBC, Cipher Block Chaining ）

  • 优点：
    • 安全性高。每个密文分组依赖于前一个密文分组，在不同位置出现的相同明文分组会生成不同密文分组，可抵御模式分析攻击。
    • 隐藏明文模式。由于分组之间有链接，CBC模式能有效隐藏明文中的重复模式，适合加密结构化数据（如图像）。
    • 是随机性加密模式。由于IV 的随机性，密文分组 Ci 都将被随机化，因此CBC的密文输出是随机化的，因此提高了安全性。
  • 缺点：
    • 有错误传播。若当前密文分组在传输时出现错误，会影响下一个密文分组的正确解密，即产生2个分组的解密错误。
    • 需要初始向量。第一个密文分组C1的计算需要一个随机的初始化矢量C0= IV。因此，对于m个明文分组，CBC将输出m+1个密文分组，增加了信道传输负荷。

3、密码反馈模式（CFB, Cipher FeedBack）

  • 优点：

    • 具有流加密特性。CFB模式将分组密码转换为流密码，且无需填充，适合加密流数据或数据长度不是分组大小整数倍的场景。

    • 隐藏明文模式。由于当前密文分组的解密依赖于前一个密文分组，在不同位置出现的相同明文分组会生成不同密文分组，可抵御模式分析攻击。

    • 是随机性加密模式。由于IV 的随机性，密文分组 Ci 都将被随机化，因此CBC 的密文输出是随机化的，因此提高了安全性。

  • 缺点

    • 有错误传播。若当前密文分组在传输时出现错误，会影响下一个密文分组的正确解密，即产生2个分组的解密错误。
    • 需要初始向量。第一个密文分组C1的计算需要一个随机的初始化矢量C0= IV。因此，对于m个明文分组，CFB将输出m+1个密文分组，增加了信道传输负荷。
    • 串行加密效率低。每个分组的加密依赖于前一个分组的密文，无法并行加密。

4、 输出反馈模式（OFB, Output FeedBack）

  • 优点：
    • 具有流加密特性。OFB模式将分组密码转换为流密码，且无需填充，适合加密流数据或数据长度不是分组大小整数倍的场景。
    • 隐藏明文模式。由于当前密文分组的解密依赖于前一组密钥流，在不同位置出现的相同明文分组会生成不同密文分组，可抵御模式分析攻击。
    • 是随机性加密模式。由于IV 的随机性，密文分组 Ci 都将被随机化，因此OFB 的密文输出是随机化的，因此提高了安全性。
    • 无错误传播。由于密钥流的生成与明文和密文无关，因此单个密文分组的错误不会影响后续密文分组的解密。
  • 缺点：
    • 需要初始向量。第一个密文分组C1的计算需要一个随机的初始化矢量C0= IV。因此，对于m个明文分组，OFB将输出m+1个密文分组，增加了信道传输负荷。
    • 串行加密效率低。当前密钥流的生成依赖于前一组密钥流，故无法并行加密。

5、计数器模式（CTR, Counter Mode）

  • 优点：
    • 具有流加密特性。CTR模式将分组密码转换为流密码，且无需填充，适合加密流数据或数据长度不是分组大小整数倍的场景。
    • 隐藏明文模式。由于计数器的值不断改变，在不同位置出现的相同明文分组会生成不同密文分组，可抵御模式分析攻击。
    • 是随机性加密模式。由于计数器的值不断改变，密文分组 Ci 都将被随机化，因此CTR的密文输出是随机化的，因此提高了安全性。
    • 无错误传播。由于密钥流的生成与明文和密文无关，单个密文分组的错误不会影响后续密文分组的解密。
    • 可采用并行加密。允许同时加密多个数据分组，适合并行处理，提升加密速度。
  • 缺点：
    • 存在重放攻击风险。若不使用额外的防重放机制，攻击者可能重放旧消息。
    • CTR只提供加密，不保证数据完整性，需结合其他机制（如HMAC）来验证。
